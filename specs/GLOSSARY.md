# Glossary

Quick reference for technical terms used across specs. Each term is explained where it first appears — this page is for lookup.

| Term | Plain meaning | Spec |
|------|---------------|------|
| **Allgard** | Superseded. Discovery and health are now built into Leden as gossip-based extensions. See `leden.discovery`. | `allgard.overview` |
| **Block-scoped** (view/borrow) | A temporary reference valid until the end of the enclosing `{ }` block. Used for fixed-layout data (struct fields, arrays). Strings are NOT block-scoped — they own heap buffers and use inline access. | `mem.borrowing` |
| **Borrow** / **View** | A temporary reference to data you don't own. The compiler tracks it so it can't outlive the data or conflict with mutations. | `mem.borrowing` |
| **Box** | A container that wraps a value and gives scoped access through `with` or inline expression access. The family: `Cell<T>` (single-task mutable), `Pool<T>`+`Handle<T>` (identity-based), `Shared<T>` (read-heavy cross-task), `Mutex<T>` (exclusive cross-task), `Owned<T>` (linear heap). | `mem.boxes` |
| **Capture** (closures) | When a closure uses a variable from outside its body, that variable is *captured* — copied or moved into the closure so it's available when the closure runs later. | `mem.closures` |
| **Desugaring** | Expanding syntactic shortcuts into their full form before the compiler processes them. Example: `a + b` becomes `a.add(b)`. | `comp.semantic-hash` |
| **Gard** | An isolated server process with its own state and lifecycle. Communicates with other gards through Leden sessions. Coarser-grained than actors — a gard is a domain, not an object. A gard is just a Leden endpoint that participates in gossip. | `leden.discovery` |
| **Exclusive access** | Only one mutable reference exists, and no other references (read or write) can coexist. Prevents data races. Sometimes called "aliasing XOR mutation." | `mem.borrowing` |
| **Linear** | A value the compiler requires you to consume exactly once — not zero times, not twice. Think of a paper ticket: the gate tears it (one consumption) and you can't hand it over again. `@resource` structs, `Owned<T>`, and linear elements in pools all follow the same L1–L6 rules. "Linear" is the academic term; "must-consume" is the plain version. Cousin concept: affine = at most once (that's what Rust's `Drop` gives you). | `mem.linear` |
| **Monomorphization** | When you call a generic function like `sort<i32>`, the compiler generates a specialized version of `sort` just for `i32`. Fast (direct calls, no runtime type checks) but increases binary size. | `type.generics` |
| **Move** | Transferring a value from one variable to another. After a move, the original variable is unusable — the compiler forbids reading it. Only applies to types larger than 16 bytes. | `mem.ownership` |
| **Must-consume** (resource types) | A type marked `@resource` that the compiler forces you to use exactly once — you can't forget to close a file or commit a transaction. See **Linear** for the shared rule set across `@resource`, `Owned<T>`, and `Pool<Linear>`. | `mem.resources` |
| **Must-use** (task handles) | `TaskHandle` must be explicitly joined (wait for result) or detached (fire-and-forget). Dropping it without doing either is a compile error. This is *affine* (at most once), not *linear* — you can always detach to drop it. Contrast with `@resource`, which is strictly linear. | `conc.async` |
| **Leden** | Standalone networking and IPC protocol. Binary, versioned, transport-agnostic. Moves bytes between endpoints. No knowledge of gards or Allgard. Separate crate. | `leden.overview` |
| **Inline access** | Temporary access to a collection element that lasts only for the expression. Used for growable sources (Vec, Pool, Map) because they might reallocate. String slices (`s[i..j]`) are also inline-only (S2). Multi-statement access uses `with...as`. | `mem.borrowing` |
| **`with` block** | First-class block scope for multi-statement access to collection elements, Cell, Shared, and Mutex values. `return`, `try`, `break`, `continue` work naturally. | `mem.borrowing` |
| **Auto-wrap** | Automatic coercion of a bare value into `T?` or `T or E` in specific positions. For `T?`: return and assignment. For `T or E`: return only (the branch is picked by the value's type, enforced by disjointness). Everywhere else the value must already have the union type. | `type.optionals`, `type.errors` |
| **Disjointness rule** | `T or E` requires T ≠ E using Rask's nominal-vs-alias distinction. Makes type-based branch disambiguation unambiguous at construction. `type X = i32` creates a distinct nominal; `type alias X = i32` is transparent. | `type.errors` |
| **`ErrorMessage`** | Structural trait requiring `func message(self) -> string`. Enforced at `T or E` type formation — every `E` must satisfy it. Richer error capabilities (lines, context, codes) layer opt-in on top. Primitives don't qualify; newtype them. | `type.errors` |
| **Narrowing** | Type refinement inside a conditional block based on a predicate over the scrutinee. Rides on `const` — only const bindings narrow. `mut` scrutinees must use `if x? as v` to bind a const in the block. No flow-typing subsystem. | `type.optionals`, `type.errors` |
| **Structural trait matching** | The compiler checks if a type has all the methods a trait requires — matching by shape, not by explicit declaration. If your type has a `compare(self, other: T) -> Ordering` method, it satisfies `Comparable` automatically. | `type.generics` |
| **`T or E`** | Builtin sum type for fallible values — compiler-generated tagged union with two branches (`T` success, `E` error). Not a user enum. No `Ok`/`Err` constructors; branch is picked by the value's type at return. Requires T ≠ E and `E: ErrorMessage`. | `type.errors` |
| **`T?`** | Builtin status type — "present `T`" or `none` (absent sentinel). Not a user enum. No `Some`/`None` constructors. Operator-only surface: `?`, `?.`, `??`, `!`, `try`, `== none`. No `match` arms. | `type.optionals` |
| **Vtable** | A table of function pointers, one per trait method. When you use `any Trait`, the runtime looks up the right function in this table and calls it. Costs one pointer indirection per call. | `type.traits` |
